# v4.1.0 Design Spec — Neon, Status & Particle Expansions

**Date:** 2026-06-25
**Version:** 4.1.0
**Type:** Minor feature release (no breaking changes)

---

## Overview

v4.1.0 adds 1 new component category (Neon) and extends 2 existing categories (Status, Particle) with 7 new components total. The library grows from 58 to 65 components across 20 categories. The demo app and Storybook are updated to cover all new components.

---

## New Components

### 1. Neon Category — `src/components/neon/` (new)

Glowing neon-style loaders using CSS box-shadow and animation. Designed for dark UIs, gaming themes, and cyberpunk aesthetics.

#### `NeonPulse`
A pulsing glowing ring that expands and fades in opacity.

Props:
- `size?: number` — diameter in px (default: 60)
- `color?: string` — neon glow color (default: `#3b82f6`)
- `glowIntensity?: 'low' | 'medium' | 'high'` — controls box-shadow spread (default: `medium`)
- `speed?: 'slow' | 'normal' | 'fast'`
- Standard `BaseLoaderProps`

#### `NeonSpinner`
A spinning arc with a glowing neon trail effect using box-shadow on a conic-gradient masked ring.

Props:
- `size?: number` — diameter in px (default: 60)
- `color?: string` — neon color (default: `#a855f7`)
- `thickness?: number` — arc stroke width in px (default: 4)
- `speed?: 'slow' | 'normal' | 'fast'`
- Standard `BaseLoaderProps`

#### `NeonText`
A text label that pulses with a neon glow, cycling through brightness states.

Props:
- `text?: string` — display text (default: `'LOADING'`)
- `color?: string` — neon glow color (default: `#22d3ee`)
- `fontSize?: number` — font size in px (default: 18)
- `speed?: 'slow' | 'normal' | 'fast'`
- Standard `BaseLoaderProps`

### 2. Status Category Extensions — `src/components/status/`

Completes the status set (currently Success + Error) with Warning and Info indicators.

#### `WarningIndicator`
An animated warning triangle with a `!` mark inside. Optionally shakes to draw attention.

Props:
- `size?: number` — icon size in px (default: 48)
- `color?: string` — triangle color (default: `#f59e0b`)
- `animate?: boolean` — whether to run draw-in animation on mount (default: `true`)
- `shake?: boolean` — shake animation (default: `false`)
- Standard `BaseLoaderProps`

#### `InfoIndicator`
An animated circle with an `i` mark that optionally pulses.

Props:
- `size?: number` — icon size in px (default: 48)
- `color?: string` — icon color (default: `#3b82f6`)
- `animate?: boolean` — whether to run draw-in animation on mount (default: `true`)
- `pulse?: boolean` — ongoing pulse glow (default: `false`)
- Standard `BaseLoaderProps`

---

### 3. Particle Category Extensions — `src/components/particle/`

Two new particle components to fill out the smallest category (currently 2 components).

#### `ParticleTrail`
Particles that follow a circular path and fade as they trail behind a leading particle.

Props:
- `size?: number` — orbit diameter in px (default: 60)
- `count?: number` — number of trailing particles (3–10, default: 6)
- `color?: string` — particle color (default: `#3b82f6`)
- `speed?: 'slow' | 'normal' | 'fast'`
- Standard `BaseLoaderProps`

#### `ParticleField`
An ambient field of floating, drifting particles that wander freely within a bounded area.

Props:
- `width?: number` — field width in px (default: 120)
- `height?: number` — field height in px (default: 120)
- `count?: number` — number of particles (5–30, default: 12)
- `color?: string` — particle color (default: `#3b82f6`)
- `speed?: 'slow' | 'normal' | 'fast'`
- Standard `BaseLoaderProps`

## Constraints

- No breaking changes — all new props have defaults
- CSS-only animations (no JS animation loops)
- All components extend `BaseLoaderProps`
- Neon effects use CSS `box-shadow` + `text-shadow` — no external deps
- `ParticleField` positions particles with inline styles using CSS animation keyframes (staggered delays), not canvas
- Each new component gets an ARIA label and optional `testId`
